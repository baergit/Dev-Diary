## 🚀 仓库目的
记录作为开发者的日常成长轨迹，包括：
- ✅ **技术笔记**：新学到的框架/工具/技巧  
- 🐞 **Debug日志**：那些让我掉头发的BUG解决方案  
- 💡 **灵感池**：临时想法与技术脑洞  
- 📈 **复盘总结**：每周/月项目经验沉淀

## ✍️ 写作规范
1. **文件命名**  
   - 每日笔记：`YYYY-MM-DD.md`（如 `2024-08-01.md`）
   - 专题文档：`[主题].md`（如 `Redis缓存策略.md`）

2. **内容格式**  
   ```markdown
   ## [主题]
   ### 1. 问题背景
   ### 2. 解决过程
   ### 3. 最终方案
   ### 4. 延伸思考
   ```

3. **标签系统**
   `#算法` `#前端` `#职场` `#踩坑记录`

## 🌟 最佳实践
